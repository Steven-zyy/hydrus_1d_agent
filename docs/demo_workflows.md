## Demo 3B: CSV Atmospheric Boundary and Material Parameters (Stable Demo)

Purpose: run a 30-day two-layer water-flow case with atmospheric forcing from CSV and direct van Genuchten material parameters from CSV. This is the stable user demo; all 30 solver steps converge.

Stable demo config:

```text
config\new_user_dynamic_csv_test.json
```

Atmospheric forcing CSV:

```text
test_inputs\new_user_dynamic_test\atmosphere_stable_30d.csv
```

Material parameter CSV:

```text
test_inputs\new_user_dynamic_test\materials_vg_stable.csv
```

Required material CSV columns:

```text
material,theta_r,theta_s,alpha_1_m,n,Ks_m_d,l
```

Unit and parameter convention:

- `material`: non-empty unique material name used by soil layers.
- `theta_r`, `theta_s`: dimensionless residual and saturated water contents.
- `alpha_1_m`: van Genuchten alpha in 1/m.
- `n`: van Genuchten n, greater than 1.
- `Ks_m_d`: saturated hydraulic conductivity in m/day.
- `l`: dimensionless pore-connectivity parameter.

Set `HYDRUS_EXE` before running:



The review summary shows both the atmospheric CSV metadata and the material CSV metadata, including record count, time range, rainfall totals, material names, and parameter values.

## Reading Run Reliability

A HYDRUS process return code of 0 means the executable process finished. It does not guarantee that the numerical solution converged. The agent checks `Error.msg` and QC outputs before assigning final reliability.

Open:

```text
runs\<case_id>\pipeline_summary.json
runs\<case_id>\report.md
runs\<case_id>\outputs\qc_summary.json
```

Important fields in `pipeline_summary.json`:

- `execution_status`: process execution status.
- `hydrus_numerical_status`: convergence status inferred from HYDRUS messages.
- `qc_status`: post-run QC status.
- `overall_status`: final status for the workflow.

If the report says HYDRUS returned code 0 but numerical failure was detected in `Error.msg`, treat the run as incomplete. Output files and figures may still be useful for diagnosis, but plots after the failure time may be unreliable. Check the atmospheric forcing, rainfall intensity, material parameters, initial condition, and numerical controls before using the results.

## Comparing Demo Results

For each demo, start with:

```text
runs\<case_id>\pipeline_summary.json
runs\<case_id>\outputs\qc_summary.json
runs\<case_id>\report.md
```

Then inspect figures:

```text
runs\<case_id>\figures\
```

Useful first checks:

- Did `pipeline_summary.json` report `ok: true`?
- Did `qc_summary.json` contain warnings?
- Did `hydrus_run.log` include `Calculations have finished successfully`?
- Were the expected water-flow figures created?
- For solute cases, were `obs_concentration_vs_time.png` and `concentration_profiles.png` created?
- For field-data comparisons, were `field_overlay_theta.png` and/or `field_overlay_head.png` created?
- For scenario batches, did `runs\<batch_id>\scenario_summary.csv` contain one row per scenario?
- For scenario comparison reports, did `runs\<batch_id>\scenario_report.md` list the best RMSE and largest flux/infiltration scenarios?

Simple one-species conservative solute generation is supported for generated cases. Official solute examples are still useful benchmark and gap-analysis cases because many include adsorption, decay, reaction chains, multiple solutes, or other chemistry beyond the current generation scope. When a copied official example writes `SoluteN.out` and concentration columns, the benchmark harness can parse those outputs, add solute summaries, and generate 11 figures including concentration time series and profiles.
